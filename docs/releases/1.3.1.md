# Version 1.3.1 Release Notes

## Release Date

April 13, 2026

## Overview

This release adds `PermissionUtils`, a runtime permission request utility based on XXPermissions, with Activity extension functions, storage permission auto-adaptation, and i18n support.

## Changes

### common-ui

- **PermissionUtils**: Runtime permission request utility based on XXPermissions, supporting dangerous permissions, storage permissions (partitioned storage adaptation for Android 10+), and special permissions (MANAGE_EXTERNAL_STORAGE). Auto-handles "Don't ask again" scenario with a dialog to guide users to settings.
- **Activity Extensions**: `requestPermission`, `requestStoragePermission`, `requestManageStoragePermission`, `isPermissionGranted`
- **i18n**: Permission dialog strings in English (default), Chinese, Traditional Chinese, and Russian

### Removed

- Old permission implementation: `LiveFragment`, `LivePermissions`, `PermissionResult`

### Dependencies

- Added XXPermissions 28.2 (`com.hjq.permissions:xxpermissions`)
- Added DeviceCompat 2.6 (`com.github.getActivity:DeviceCompat`)

### app

- Added `PermissionDemoActivity` demonstrating various permission request scenarios

## Migration from 1.3.0

### Breaking Changes

- If you were using the old `LivePermissions` / `LiveFragment` / `PermissionResult` classes, migrate to `PermissionUtils` or the Activity extension functions.

### Recommended

- Use Activity extension functions (`requestPermission`, `requestStoragePermission`) for concise permission handling.
- Use `PermissionUtils.requestStorage()` for automatic Android version adaptation of storage permissions.
