# Version 1.0.5 Release Notes

## Release Date

March 17, 2026

## Overview

This release upgrades the base Activity and Fragment architecture to resolve ViewBinding and ViewModel types automatically, adds `SavedStateHandle` support for screen state restoration, and ships a new Fragment demo to verify the flow end to end.

## Highlights

### Base Architecture
- Added generic ViewBinding and ViewModel class resolution for `BaseActivity` and `BaseFragment`
- Added automatic ViewModel creation with `SavedStateHandle` support
- Added `NoViewModel` for screens that only need binding and lifecycle helpers

### Navigation and State
- Replaced the legacy Activity result helper with Activity Result API registration helpers
- Added a Fragment demo screen that keeps ViewModel state across Activity recreation
- Added unit and instrumentation coverage for ViewModel resolution and saved state restoration

### UI and Demo App
- Wrapped `BaseComposeActivity` content in `AppTheme`
- Added Fragment demo entry to the sample app
- Fixed the sample Glide module package to match the current app namespace

### All Modules

## Migration Notes

- `BaseActivity` subclasses should declare both binding and ViewModel generic types.
- `BaseFragment` subclasses should declare both binding and ViewModel generic types.
- Use `NoViewModel` where a screen does not need a concrete ViewModel.
- Migrate old `startActivityForResult` helper usage to `registerStartActivityForResult` and `launchActivity`.
